## Motivation

No near-term — and arguably no long-term — useful computation is purely quantum.
Quantum processors will be accelerators within classical systems, much as GPUs are
today: invoked for the sub-tasks they accelerate, embedded in classical data
pipelines, orchestration, and user interfaces. This chapter studies the
*architecture* of such hybrid systems, the very pattern this repository's API
embodies.

## Theory

A hybrid quantum-classical system partitions work by comparative advantage. The
quantum processor prepares states, evolves them, and measures; the classical
processor handles optimisation, data management, error mitigation, and control
(Chapters 7, 17). The interface between them — latency, bandwidth, and the cost of
moving between quantum and classical representations — is often the true
performance bottleneck, not the quantum hardware itself. Tight feedback loops, as
in real-time decoding (Chapter 8), demand co-located classical compute; loose
loops, as in molecular screening (Chapter 10), tolerate cloud round-trips.

A deeper distinction governs where quantum advantage is most plausible: the nature
of the *data*. **Classical data** loaded into a quantum computer must pass through
a state-preparation bottleneck, and the dequantisation results of Chapter 16
caution against expecting generic speedups. **Quantum data** — states produced by
a quantum sensor, simulator, or experiment — never leave the quantum domain, and
learning directly on such data is where theory most strongly supports advantage:
a quantum model can sometimes characterise a quantum process from exponentially
fewer experiments than any classical strategy. Designing hybrid systems thus
begins with asking where the data come from.

Engineering maturity matters as much as theory. A production hybrid service needs
validated inputs, graceful degradation when the quantum backend is unavailable,
reproducible environments, observability, and clear API contracts. The companion
service in this repository demonstrates these patterns: thin, validated web routes
wrap thoroughly tested library functions, so the science is verifiable
independently of the deployment, and classical simulation provides a faithful
fallback during development.

## From theory to code

The FastAPI service is the hybrid architecture in miniature: each endpoint
validates a request, dispatches to a tested `aiquantum` function, and returns a
typed response.

```python
from fastapi.testclient import TestClient
from aiquantum.api.app import app

client = TestClient(app)
r = client.post("/api/v1/qml/kernel", json={"x_a": [[0.0], [0.0]]})
print(r.json()["kernel_matrix"])   # 2x2 quantum kernel
```

## Summary

Quantum processors are accelerators within classical systems. The interface and
the *origin of the data* — classical versus quantum — determine where advantage
lives, and sound software engineering determines whether it can be delivered.
