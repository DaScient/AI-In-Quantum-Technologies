## Motivation

Both quantum mechanics and machine learning are, computationally, linear algebra
performed at scale and differentiated for optimisation. A shared toolbox lets us
move between the two without friction. This chapter is deliberately practical: it
is the reference the later chapters assume.

The objects are familiar — vectors, matrices, eigenvalues, gradients — but their
quantum use demands care. A matrix that is almost Hermitian may produce complex
“probabilities.” A reconstructed density matrix with a tiny negative eigenvalue
may be a harmless statistical artefact or evidence of a flawed pipeline. A
state-vector simulator may produce exact amplitudes for twenty qubits, yet the
same circuit on hardware yields only samples. Machine learning adds another
layer: gradients, loss landscapes, regularisation, and numerical conditioning.
The goal is not mathematical ornamentation; it is reliable computation.

## Theory

### Complex linear algebra

**Complex linear algebra** is the native language of quantum theory. State
vectors live in $\mathbb{C}^d$, inner products use conjugate transposes, and
operators are complex matrices. For vectors $|u\rangle$ and $|v\rangle$,

$$
\langle u|v\rangle = \sum_j u_j^* v_j.
$$

The conjugation is not cosmetic: it guarantees non-negative norms and makes
unitary evolution norm-preserving. A matrix $U$ is **unitary** when
$U^\dagger U=I$. Unitaries represent closed-system time evolution and ideal
quantum gates. A matrix $H$ is **Hermitian** when $H=H^\dagger$; Hermitian
operators represent observables and Hamiltonians.

Several matrix norms recur in analysis. The Frobenius norm is convenient for
least-squares fitting; the spectral norm controls worst-case vector stretching;
the trace norm appears in state distinguishability. Numerical code should choose
norms according to the operational question, not merely convenience.

### Spectral theorem and matrix functions

The **spectral theorem** states that every Hermitian operator has an orthonormal
eigenbasis and real eigenvalues:

$$
H = \sum_k \lambda_k |k\rangle\langle k|.
$$

This theorem underpins measurement, diagonalisation of Hamiltonians, and the
matrix functions used for fidelity, entropy, and dynamics. If $f$ is a scalar
function, then

$$
f(H)=\sum_k f(\lambda_k)|k\rangle\langle k|.
$$

The matrix exponential

$$
U(t)=e^{-iHt}
$$

generates closed-system dynamics. For small matrices, direct diagonalisation or
`scipy.linalg.expm` is often adequate. For large sparse Hamiltonians, Krylov
methods, Trotterisation, tensor networks, or problem-specific structure become
necessary. No method removes the exponential worst case for arbitrary quantum
systems; claims of speed must always specify the structure being exploited.

The logarithm and square root of positive operators also appear frequently. Von
Neumann entropy uses $\log\rho$; mixed-state fidelity uses $\sqrt{\rho}$. These
operations are stable only when eigenvalues are handled carefully. Tiny negative
values caused by floating-point noise should not be passed blindly into a square
root or logarithm.

### Tensor products and scaling

The **tensor product** combines systems. If $A$ acts on system $1$ and $B$ acts
on system $2$, then $A\otimes B$ acts on the joint space. Dimensions multiply:
$n$ qubits have dimension $2^n$, and a dense operator on them has $4^n$ complex
entries. This fact explains why even simple-looking code can become impossible at
scale.

Tensor products also encode locality. A two-qubit gate embedded in a larger
register is an identity on untouched qubits tensored with a non-trivial operator
on the target subsystem, possibly after index permutation. Efficient simulators
exploit this locality rather than materialising every full matrix. Machine
learning models should exploit analogous structure through local descriptors,
graphs, tensor networks, sparse kernels, or equivariant architectures.

A common computational blueprint is:

```text
local operators ──tensor/embed──> global observable
       │                              │
       v                              v
 sparse or structured algebra ──> expectation / loss / gradient
```

The danger is accidental densification: constructing a $2^n\times2^n$ matrix
when a sequence of local contractions would suffice.

### Pauli basis and operator decomposition

The single-qubit Pauli matrices $I,X,Y,Z$ form an orthogonal basis for
$2\times2$ matrices under the Hilbert-Schmidt inner product
$\langle A,B\rangle=\operatorname{Tr}(A^\dagger B)$. Tensor products of Paulis
form a basis for $n$-qubit operators. Any $n$-qubit Hamiltonian can be written as

$$
H = \sum_j c_j P_j,
$$

where each $P_j$ is a Pauli string such as $Z_0X_1Y_3$. This representation is
central to variational algorithms, measurement grouping, error models, and
Hamiltonian learning. The `aiquantum.linalg` helper `n_qubit_paulis` provides the
basis for small explicit calculations; it is not a license to enumerate huge
bases without considering $4^n$ scaling.

Pauli decompositions are also a bridge between theory and hardware. A simulator
can compute $\langle H\rangle$ directly from a state vector. A device estimates
$\langle P_j\rangle$ by measuring compatible Pauli strings over many shots, then
combines them classically. The same formula is used, but the statistical and
operational costs are different.

### Probability, statistics, and finite shots

Quantum measurement is intrinsically probabilistic, so estimation theory is
unavoidable. Finite sampling turns exact expectation values into noisy estimates
whose uncertainty often scales as $1/\sqrt{N}$ in the number of shots $N$. The
binary entropy function

$$
H_2(p) = -p\log_2 p - (1-p)\log_2(1-p)
$$

recurs in error correction and cryptography (Chapters 8 and 13). Confidence
intervals, hypothesis tests, Bayesian updates, and bootstrap methods are not
optional extras when claims rest on measured data.

The distinction between simulator and hardware should be explicit in every
workflow:

- **Classical simulation** can expose amplitudes, exact probabilities, and
  noiseless gradients for systems small enough to fit in memory.
- **Quantum hardware** produces samples subject to shot noise, decoherence,
  calibration drift, crosstalk, and readout error.
- **Noisy simulation** is an intermediate model: useful for design and testing,
  but only as accurate as its noise assumptions.

### Positive-semidefinite projection

A valid density matrix is Hermitian, positive semi-definite, and trace one.
Tomography, finite sampling, and unconstrained optimisation can produce estimates
that violate these conditions. **Positive-semidefinite projection** repairs an
estimate by symmetrising it, diagonalising it, projecting eigenvalues onto the
probability simplex, and reconstructing the matrix. Conceptually:

```text
raw estimate -> Hermitian part -> eigenvalues/eigenvectors
                    │                    │
                    v                    v
              simplex projection -> PSD, trace-one density matrix
```

Projection is a numerical safeguard, not a substitute for good experimental
design. If projection makes a tiny correction, it may simply remove sampling
noise. If it radically changes the estimate, the measurement model, calibration,
or reconstruction procedure deserves scrutiny.

### Distance and fidelity measures

Quantum states can be compared in several ways. **Fidelity** measures overlap and
has an operational interpretation in state discrimination and benchmarking. For a
pure state and a mixed state,

$$
F(|\psi\rangle,\rho)=\langle\psi|\rho|\psi\rangle.
$$

Trace distance quantifies distinguishability under optimal measurement. Hilbert-
Schmidt distance is easy to compute but less directly operational. Bures distance
is derived from fidelity and respects quantum state geometry. The correct measure
depends on the task: training loss, certification, visualisation, and hardware
benchmarking may require different notions of closeness.

### Optimisation and automatic differentiation

Training a model and shaping a control pulse are both optimisation problems.
Gradient-based methods dominate because they scale better than exhaustive search
in high-dimensional parameter spaces. **Automatic differentiation** computes
derivatives of programs by systematically applying the chain rule. In classical
neural networks this is backpropagation through tensor operations. In variational
quantum algorithms, gradients may be computed by parameter-shift rules, adjoint
methods, finite differences, or hardware-compatible stochastic estimators.

A hybrid loop has the form:

```text
parameters θ -> quantum circuit / simulator -> measured expectation
       ^                                         │
       │                                         v
       └──────── classical optimiser <── loss and gradient
```

On a simulator, gradients may be exact up to floating-point error. On hardware,
expectations and gradients are estimated from shots, so optimisation must handle
noise. Barren plateaus, ill-conditioning, and local minima are not abstract
pathologies; they determine whether a variational workflow is practical.

### Numerical hygiene

**Numerical hygiene** is the discipline of preventing artefacts from masquerading
as physics. Useful habits include:

- symmetrise matrices before treating them as Hermitian;
- check $U^\dagger U\approx I$ before calling a matrix a gate;
- clip or project eigenvalues only with a documented tolerance;
- avoid forming dense tensor products unless the size is intentionally small;
- seed pseudo-random examples used in documentation and tests;
- distinguish absolute tolerance from relative tolerance in comparisons.

These practices are mundane, but they keep later AI conclusions from resting on
unstable linear algebra.

## From theory to code

`aiquantum.linalg` collects the primitives. Two are worth highlighting: the
physicality projection and the fidelity.

```python
import numpy as np
from aiquantum.linalg import project_to_density_matrix, is_density_matrix, fidelity

raw = np.array([[1.2, 0.0], [0.0, -0.2]], dtype=complex)  # unphysical estimate
rho = project_to_density_matrix(raw)
print(is_density_matrix(rho))                              # True
print(fidelity(rho, np.array([[1, 0], [0, 0]], dtype=complex)))
```

The projection clips negative eigenvalues and renormalises — the trace-preserving
algorithm of Smolin, Gambetta, and Smith — guaranteeing a valid state to feed
downstream models.

A small Pauli-basis calculation can be written explicitly. For larger systems,
use structured representations rather than enumerating all $4^n$ basis elements:

```python
import numpy as np

paulis = {
    "I": np.array([[1, 0], [0, 1]], dtype=complex),
    "X": np.array([[0, 1], [1, 0]], dtype=complex),
    "Y": np.array([[0, -1j], [1j, 0]], dtype=complex),
    "Z": np.array([[1, 0], [0, -1]], dtype=complex),
}

observable = 0.7 * paulis["Z"] + 0.2 * paulis["X"]
coefficients = {name: np.trace(P.conj().T @ observable) / 2 for name, P in paulis.items()}
print(coefficients)
```

Qiskit can check whether a proposed matrix is unitary before it is treated as a
gate in a circuit model:

```python
import numpy as np
from qiskit.quantum_info import Operator

u = np.array([[1, 1], [1, -1]], dtype=complex) / np.sqrt(2)
op = Operator(u)
print(op.is_unitary())
```

PennyLane illustrates automatic differentiation through a quantum expectation.
This example uses a simulator; hardware gradients require shot-based estimation
or device-supported gradient rules.

```python
import pennylane as qml
from pennylane import numpy as pnp

dev = qml.device("default.qubit", wires=1)

@qml.qnode(dev)
def energy(theta):
    qml.RY(theta, wires=0)
    return qml.expval(qml.PauliZ(0))

grad_energy = qml.grad(energy)
theta = pnp.array(0.3, requires_grad=True)
print(energy(theta))
print(grad_energy(theta))
```

Cirq provides another compact way to reason about unitary matrices associated
with circuit fragments:

```python
import cirq
import numpy as np

q = cirq.LineQubit(0)
circuit = cirq.Circuit(cirq.H(q), cirq.Z(q))
u = cirq.unitary(circuit)
print(np.allclose(u.conj().T @ u, np.eye(2)))
```

Each snippet is intentionally small enough for exact classical computation. The
same algebra scales into later chapters, but the implementation strategy changes
once dense matrices no longer fit in memory or hardware sampling replaces exact
state access.

## Summary

A small, well-understood numerical toolbox carries the entire book. Complex
linear algebra defines states and gates; the spectral theorem enables dynamics,
entropy, and fidelity; tensor products explain both locality and exponential
scaling; Pauli decompositions connect Hamiltonians to measurement; and automatic
differentiation drives optimisation. Disciplined numerical hygiene is the shared
language of quantum simulation and machine learning, and it is essential for
separating physical conclusions from computational artefacts.
