## Motivation

Both quantum mechanics and machine learning are, computationally, linear algebra
performed at scale and differentiated for optimisation. A shared toolbox lets us
move between the two without friction. This chapter is deliberately practical: it
is the reference the later chapters assume.

## Theory

**Linear algebra.** States are vectors, operations are unitary matrices, and
observables are Hermitian matrices. The spectral theorem — every Hermitian
operator has a real eigenbasis — underpins measurement, diagonalisation of
Hamiltonians, and the matrix functions (square root, logarithm, exponential) we
use for fidelity, entropy, and time evolution. The matrix exponential
$U = e^{-iHt}$ generates dynamics; computing it stably for large systems is a
recurring challenge addressed by Krylov and tensor-network methods.

**Probability and statistics.** Quantum measurement is intrinsically
probabilistic, so estimation theory is unavoidable. Finite sampling turns exact
expectation values into noisy estimates whose error scales as $1/\sqrt{N}$ in the
number of shots $N$. The binary entropy function $H_2(p) = -p\log_2 p -
(1-p)\log_2(1-p)$ recurs in error correction and cryptography (Chapters 8 and
13).

**Optimisation and automatic differentiation.** Training a model and shaping a
control pulse are both optimisation problems. Gradient-based methods dominate,
and modern frameworks compute gradients automatically by backpropagation. The
GRAPE algorithm of Chapter 7 is gradient ascent on a fidelity landscape;
variational quantum algorithms (Chapter 17) are gradient descent through a hybrid
quantum-classical loop. Understanding when these landscapes are benign — and when
they suffer barren plateaus — is essential.

**Numerical care.** Density matrices estimated from data may drift outside the
physical set; we restore positivity and unit trace by projection onto the
probability simplex of eigenvalues. Angles must be wrapped; small denominators
must be guarded. These are not pedantry but the difference between a result and
an artefact.

## From theory to code

`aiquantum.linalg` collects the primitives. Two are worth highlighting: the
physicality projection and the fidelity.

```python
import numpy as np
from aiquantum.linalg import project_to_density_matrix, is_density_matrix, fidelity

raw = np.array([[1.2, 0.0], [0.0, -0.2]], dtype=complex)  # unphysical estimate
rho = project_to_density_matrix(raw)
print(is_density_matrix(rho))                              # True
print(fidelity(rho, np.array([[1, 0], [0, 0]], dtype=complex)))
```

The projection clips negative eigenvalues and renormalises — the trace-preserving
algorithm of Smolin, Gambetta, and Smith — guaranteeing a valid state to feed
downstream models.

## Summary

A small, well-understood numerical toolbox carries the entire book. Spectral
methods, sampling statistics, gradient optimisation, and disciplined numerical
hygiene are the shared language of quantum simulation and machine learning.
