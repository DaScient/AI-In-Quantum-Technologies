## Motivation

Quantum machine learning (QML) asks whether quantum computers can learn from data
in ways classical computers cannot. The cleanest formulation maps classical data
into quantum states via a **feature map** and measures their overlaps as a
**kernel**, connecting QML to the well-understood theory of kernel methods and
support vector machines. This chapter develops that bridge and tempers it with a
sober view of when advantage is plausible.

## Theory

A quantum feature map $x \mapsto |\phi(x)\rangle$ embeds a classical input into a
Hilbert space by preparing a data-dependent state, typically through
parameterised rotations — *angle encoding*. The induced **quantum kernel** is the
squared overlap

$$
k(x, y) = \left|\langle \phi(x) \mid \phi(y)\rangle\right|^2,
$$

a positive semi-definite similarity measure that can be plugged into any
kernel-based learner. The hope is that a quantum feature map reaches a space that
is hard to compute classically yet useful for the task, yielding a genuine
advantage. Havlíček and co-workers formalised this kernel viewpoint and
demonstrated it on hardware.

The caveats are essential and the textbook insists on them. First, expressivity
is not utility: a feature map so rich that all states are nearly orthogonal makes
the kernel a glorified identity matrix and generalisation collapses. Second,
estimating kernel entries on hardware costs many shots, and the statistical noise
can erase any theoretical edge. Third — and decisively — recent *dequantisation*
results show that several proposed quantum learners can be matched by classical
algorithms once data access is accounted for. The honest position is that quantum
advantage in machine learning is possible for carefully structured problems,
likely those with quantum-native data (Chapter 18), but is not a generic
consequence of using a quantum computer.

## From theory to code

`aiquantum.qml` computes exact quantum kernels by state-vector simulation,
backing `/api/v1/qml/kernel`:

```python
import numpy as np
from aiquantum.qml import quantum_kernel_matrix

x = np.array([[0.0], [0.0], [np.pi / 2]])
k = quantum_kernel_matrix(x)
print(k.round(3))   # identical inputs -> 1; orthogonal angle -> 0
```

## Summary

Quantum kernels recast QML as kernel learning with quantum-computed similarities.
The framework is elegant, but advantage is conditional — defeated by trivial or
overly expressive maps, by shot noise, and by classical dequantisation.
