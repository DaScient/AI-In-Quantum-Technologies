## Motivation

Quantum machine learning (QML) asks whether quantum processors can learn from data in ways classical processors cannot. The most disciplined entry point is not to call every parameterised circuit a neural network, but to start from a mature classical theory: kernel learning. A **feature map** embeds an input into a space where simple decision rules may become powerful. In classical machine learning that space may be an implicit reproducing-kernel Hilbert space; in QML it may be the Hilbert space of $n$ qubits.

The quantum-kernel programme is attractive because it isolates a concrete question. Given classical examples $x_i$, prepare quantum states $|\phi(x_i)\rangle$ and estimate their overlaps. These overlaps form a Gram matrix that can be passed to a conventional support vector machine (SVM), ridge regressor, Gaussian process, or other kernel method. The learning algorithm remains classical and auditable; the quantum device is used only to evaluate a similarity function that may be difficult to compute classically.

That separation also makes the caveats unavoidable. If the kernel is easy to simulate, the quantum processor is unnecessary. If the feature map is too expressive, most states become nearly orthogonal and the kernel becomes uninformative. If the entries require too many hardware shots to estimate accurately, the cost of the quantum subroutine can dominate the entire pipeline. The honest scientific question is therefore not whether QML sounds powerful, but whether a particular data source, feature map, and learning task create useful structure that is hard for classical algorithms to reproduce.

## Theory

### Feature maps as state preparation

A **quantum feature map** is a data-dependent state-preparation circuit

$$
x \mapsto |\phi(x)\rangle = U_\phi(x)|0\rangle^{\otimes n}.
$$

For classical tabular data, the most common construction is **angle encoding**: numerical features are rescaled into rotation angles. A one-feature, one-qubit example is

$$
|\phi(x)\rangle = R_y(x)|0\rangle = \cos\frac{x}{2}|0\rangle + \sin\frac{x}{2}|1\rangle.
$$

With multiple features one can assign features to separate qubits, repeat features across layers, or entangle qubits after local rotations. Entangling gates are not decorative; they determine which cross-feature interactions the state can represent. A circuit containing only independent single-qubit rotations produces a kernel that factorises across features. A circuit with data-dependent entangling phases can create higher-order correlations analogous to polynomial feature interactions in classical kernels.

A useful engineering analogy is a learned embedding service. The raw vector $x$ is not the object used by the downstream classifier. Instead, it is converted into a representation whose geometry defines which examples are close. In QML, the geometry is the projective geometry of quantum states, and global phase is irrelevant: $|\phi\rangle$ and $e^{i\alpha}|\phi\rangle$ encode the same physical state.

### Quantum kernels

The **quantum kernel** induced by a feature map is

$$
k(x,y)=\left|\langle\phi(x)|\phi(y)\rangle\right|^2.
$$

It is a fidelity: the probability that a state prepared as $|\phi(y)\rangle$ passes a projective test for $|\phi(x)\rangle$. Operationally, one may estimate it by preparing $U_\phi(y)|0\rangle$, applying $U_\phi(x)^\dagger$, and measuring the probability of returning to $|0\rangle^{\otimes n}$:

```text
|0...0> ── U_phi(y) ── U_phi(x)† ── measure all zeros
                         |
                         +-- probability = |<phi(x)|phi(y)>|^2
```

For any finite dataset $\{x_i\}_{i=1}^m$, the matrix $K_{ij}=k(x_i,x_j)$ is positive semi-definite. One way to see this is to use the density matrix $\rho(x)=|\phi(x)\rangle\langle\phi(x)|$. Then

$$
k(x_i,x_j)=\operatorname{Tr}[\rho(x_i)\rho(x_j)],
$$

which is an inner product in the Hilbert-Schmidt space of operators. For any coefficients $c_i$,

$$
\sum_{i,j} c_i c_j k(x_i,x_j)
= \operatorname{Tr}\left[\left(\sum_i c_i\rho(x_i)\right)^2\right] \ge 0
$$

for real $c_i$ and Hermitian $\sum_i c_i\rho(x_i)$. This is exactly the condition required by kernel methods.

### Connection to SVMs and kernel methods

A **support vector machine** learns a decision function of the form

$$
f(x)=\operatorname{sign}\left(\sum_i \alpha_i y_i k(x_i,x)+b\right),
$$

where $x_i$ are training examples, $y_i\in\{-1,1\}$ are labels, and the coefficients $\alpha_i$ are obtained by a convex classical optimisation problem. The quantum processor does not train the SVM; it supplies entries of $K$. This matters because the statistical learning theory of margins, regularisation, cross-validation, and model selection still applies.

The kernel matrix is therefore a contract between physics and machine learning:

- The quantum circuit defines the hypothesis class through state geometry.
- The classical optimiser selects a separating hyperplane in the induced feature space.
- Generalisation depends on the alignment between the kernel and labels, not on circuit complexity alone.

A quantum kernel that is difficult to compute but poorly aligned with the target function is useless. Conversely, a simple kernel that captures the data-generating structure may outperform a highly entangling circuit.

### Expressivity versus utility

**Expressivity** measures how broadly a circuit family can explore Hilbert space. It is tempting to maximise it, but kernel learning rewards useful inductive bias, not arbitrary coverage. In a Hilbert space of dimension $2^n$, random states are almost orthogonal with high probability. If $|\langle\phi(x_i)|\phi(x_j)\rangle|^2\approx 0$ for all distinct examples, the kernel matrix approaches the identity. Such a matrix memorises training points but provides little smoothness for interpolation or classification.

Good feature maps are structured. They may encode symmetries of molecular Hamiltonians, periodicity of phases, locality of sensor arrays, or conserved quantities in a physical process. This mirrors classical practice: convolutional networks are useful not because they are maximally expressive, but because translation structure is built into them. Quantum feature maps need the same discipline.

### Hardware estimation and shot noise

On a state-vector simulator, $k(x,y)$ can be computed exactly up to floating-point precision. On real hardware, it is estimated from repeated measurements. If $\hat{k}$ is the fraction of all-zero outcomes after the overlap circuit, it is a binomial estimator of $k$. Its uncertainty decreases with the number of shots, but more shots mean more queue time, more calibration drift, and higher cost. Hardware noise also biases the estimate: decoherence, crosstalk, readout error, and imperfect inverses can reduce observed fidelities even when the ideal kernel is well behaved.

This distinction is essential. Simulation is a development and validation tool. Hardware execution is an experimental procedure whose results include statistical and device noise. A production QML pipeline should track shot counts, confidence intervals, backend metadata, and any error-mitigation procedure used to obtain a kernel matrix.

### Dequantisation caveats

Several QML proposals have faced **dequantisation**: classical algorithms were later found that reproduce the claimed speedups under comparable assumptions about data access, sampling, or low-rank structure. These results do not prove that all QML is impossible. They do show that speedups based only on loading classical vectors into amplitudes, or on informal appeals to exponential Hilbert-space dimension, are fragile. Any claimed advantage must specify the input model, the output required, the accuracy tolerance, and the best known classical competitor.

The most plausible opportunities are structured tasks where the kernel is hard to estimate classically and well matched to the target, or tasks involving quantum-native data that need not be classically loaded at all (Chapter 18). The correct stance is conditional optimism: quantum kernels are a rigorous framework for experimentation, not a blanket replacement for classical kernels.

## From theory to code

`aiquantum.qml` computes exact quantum kernels by state-vector simulation,
backing `/api/v1/qml/kernel`:

```python
import numpy as np
from aiquantum.qml import quantum_kernel_matrix

x = np.array([[0.0], [0.0], [np.pi / 2]])
k = quantum_kernel_matrix(x)
print(k.round(3))   # identical inputs -> 1; orthogonal angle -> 0
```

The library block above is the canonical offline reference for this repository. It deliberately uses state-vector simulation, so it is deterministic and suitable for tests. The same mathematical object can be sketched in common quantum frameworks.

### PennyLane quantum-kernel sketch

```python
import pennylane as qml
import numpy as np

n_qubits = 2
dev = qml.device("default.qubit", wires=n_qubits, shots=None)


def feature_map(x):
    for wire, angle in enumerate(x):
        qml.RY(angle, wires=wire)
    qml.CNOT(wires=[0, 1])
    qml.RZ(x[0] * x[1], wires=1)


@qml.qnode(dev)
def overlap(x, y):
    feature_map(y)
    qml.adjoint(feature_map)(x)
    return qml.probs(wires=range(n_qubits))


def quantum_kernel(x, y):
    return overlap(x, y)[0]  # probability of measuring |00>

x = np.array([0.2, 0.7])
y = np.array([0.3, 0.4])
print(float(quantum_kernel(x, y)))
```

With `shots=None`, PennyLane returns an exact simulator probability. With finite shots on a hardware-like device, the returned value becomes an estimate and should be treated statistically.

### Qiskit `FidelityQuantumKernel` sketch

```python
import numpy as np
from qiskit.circuit.library import ZZFeatureMap
from qiskit_machine_learning.kernels import FidelityQuantumKernel

feature_map = ZZFeatureMap(feature_dimension=2, reps=2, entanglement="linear")
kernel = FidelityQuantumKernel(feature_map=feature_map)

x_train = np.array([[0.1, 0.2], [0.4, 0.8], [1.0, 1.2]])
kernel_matrix = kernel.evaluate(x_vec=x_train)
print(kernel_matrix)
```

This sketch relies on `qiskit-machine-learning` to construct fidelity circuits and evaluate the Gram matrix. In real experiments, the fidelity primitive may target a simulator or hardware backend; the interpretation of the result changes accordingly.

## Summary

Quantum kernels recast QML as kernel learning with quantum-computed similarities. The feature map $U_\phi(x)$ defines the geometry, the kernel $k(x,y)=|\langle\phi(x)|\phi(y)\rangle|^2$ supplies a positive semi-definite Gram matrix, and a classical learner such as an SVM performs the optimisation. The framework is elegant and experimentally useful, but advantage is conditional: it can be defeated by trivial maps, overly expressive maps, shot noise, hardware bias, and classical dequantisation.
