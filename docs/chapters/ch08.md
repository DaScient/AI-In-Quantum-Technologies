## Motivation

Fault-tolerant quantum computing rests on **quantum error correction** (QEC): encoding a logical qubit across many physical qubits so that errors can be detected and reversed without measuring the stored quantum information directly. The computational bottleneck is **decoding** — inferring the most likely error from syndrome measurements in real time. Machine learning is relevant because decoding is a structured inference problem under uncertainty, but learned decoders must be judged against strong classical baselines, hardware latency constraints, and changing noise processes.

Near-term devices also use **error mitigation**, which is deliberately different from correction. Mitigation does not create a protected logical qubit. Instead, it estimates expectation values of ideal circuits from noisy data, often by executing related circuits and post-processing the results. This distinction matters throughout the NISQ era: mitigation can improve estimates for modest circuits, while scalable fault tolerance requires correction with repeated syndrome extraction and active recovery or Pauli-frame tracking.

## Theory

### Stabiliser formalism

A stabiliser code is defined by an abelian group $\mathcal{S}$ of Pauli operators that does not contain $-I$. The **code space** is the simultaneous $+1$ eigenspace of all stabilisers:

$$
S_i |\psi_L\rangle = |\psi_L\rangle \qquad \text{for all } S_i\in\mathcal{S}.
$$

The encoded state $|\psi_L\rangle$ is not learned by measuring the stabilisers. Because the stabilisers commute with the logical information, their measurement reveals only whether an error has moved the system out of the code space. If an error $E$ anticommutes with stabiliser $S_i$, then

$$
S_i E |\psi_L\rangle = -E S_i |\psi_L\rangle = -E |\psi_L\rangle,
$$

so the corresponding measurement outcome flips from $+1$ to $-1$. The binary list of flipped checks is the **syndrome**. A decoder maps syndrome data to a recovery operation $R$ such that $RE$ acts trivially on the encoded information, up to stabilisers.

Two errors with the same syndrome are not necessarily equivalent. Their product may be a stabiliser, in which case they have the same effect on the logical state, or it may be a logical operator, in which case choosing the wrong recovery causes a logical failure. Decoding is therefore not merely error detection; it is probabilistic inference over equivalence classes.

### Repetition code

The bit-flip **repetition code** stores one classical bit of quantum information against $X$ errors by encoding

$$
|0_L\rangle = |000\cdots 0\rangle, \qquad |1_L\rangle = |111\cdots 1\rangle.
$$

For $n$ data qubits, parity checks $Z_iZ_{i+1}$ detect neighbouring disagreements. In a three-qubit code the checks are $Z_0Z_1$ and $Z_1Z_2$. A bit flip on the middle qubit changes both parities, giving syndrome $[1,1]$ if `1` denotes a detection event. A flip on an edge qubit changes only one parity.

For independent bit-flip noise with probability below one half, a minimum-weight decoder chooses the lowest-weight error consistent with the syndrome. On a one-dimensional repetition code, this can be implemented by simple prefix logic or by a lookup table for small $n$. A **syndrome lookup decoder** enumerates possible low-weight errors, records their syndromes, and chooses a representative correction. Lookup decoding is transparent and fast for small examples, but it grows rapidly with code size and noise complexity.

The repetition code protects against one error type. A full quantum memory must handle both $X$ and $Z$ errors without cloning the state. That requirement motivates CSS codes, colour codes, and the surface code.

### Surface codes and syndrome extraction

The **surface code** arranges data qubits on a two-dimensional lattice and measures local stabilisers. Plaquette-like checks detect $Z$-type errors; star-like checks detect $X$-type errors. The locality is crucial: checks involve only neighbouring qubits, making the code compatible with planar hardware layouts.

Syndrome extraction uses ancilla qubits. An ancilla is prepared, entangled with a small set of data qubits by CNOT or controlled-phase gates, and measured. The measurement outcome is a parity check, not a direct measurement of the logical state. Repeating this procedure over time produces a three-dimensional decoding graph: two spatial dimensions plus a time dimension that captures measurement errors.

A text sketch for a single parity-check measurement is:

```text
Qiskit-style parity check for Z0 Z1 using one ancilla

q0:  |data> ----■-------------
                 |
q1:  |data> ---- | ----■-------
                 |     |
a0:  |0>   --H---X-----X---H---M  -> syndrome bit

Classical simulation: sample this circuit with an assumed noise model.
Hardware execution: run many shots on a device; outcomes include gate,
measurement, leakage, crosstalk, and drift effects.
```

The precise gate orientation depends on whether one measures an $X$-type or $Z$-type stabiliser and on the convention used for controls and targets. The invariant idea is parity extraction through an ancilla that is measured and reset.

### Minimum-weight decoding

For surface-code phenomenological noise, detection events are vertices in a graph. Edges represent possible physical faults, weighted by negative log-likelihood. **Minimum-weight perfect matching** (MWPM) pairs detection events so that the total weight is minimal. This is a powerful baseline because it is grounded in the geometry and probability model of the code.

MWPM is not the only decoder. Union-find decoders trade some optimality for speed and simplicity. Belief propagation is natural for sparse graphical models but needs care with degeneracy and short cycles. Tensor-network decoders can be accurate for certain noise structures but may be computationally heavier. The central engineering question is not “which decoder is fashionable?” but which decoder meets logical-error, latency, memory, and calibration requirements for the device at hand.

### Neural decoders

**Neural decoders** treat decoding as supervised learning, reinforcement learning, or sequence modelling. Inputs may be a single syndrome round, a history of detection events, or analogue measurement information before thresholding. Outputs may be a recovery, a correction class, or an update to a Pauli frame.

Machine learning can help when:

- Noise is spatially or temporally correlated.
- Analogue readout carries useful confidence information.
- The hardware graph has irregular defects or nonuniform error rates.
- A decoder can be trained on realistic simulation and calibrated experimental data.

The caveats are equally important. A neural decoder trained on one noise distribution may degrade when the device drifts. Training labels can be expensive for large codes. In a fault-tolerant architecture, the decoder must operate within a strict real-time budget. For these reasons, learned decoders are best framed as candidates to benchmark against MWPM, union-find, and belief-propagation methods, not as automatic replacements.

### Error mitigation versus correction

**Error correction** uses redundancy, repeated syndrome extraction, and recovery to preserve logical information during a computation. If scaled below threshold with sufficient overhead, it can in principle support arbitrarily long computations.

**Error mitigation** does not identify and repair physical errors during the circuit. It estimates ideal expectation values from noisy experiments. Common methods include:

- **Zero-noise extrapolation** (ZNE): run circuits at several effective noise levels and extrapolate an observable to the zero-noise limit.
- **Probabilistic error cancellation** (PEC): represent ideal operations as signed combinations of noisy implementable operations and reweight measurement outcomes.
- **Readout-error mitigation**: estimate and invert a measurement-confusion model.
- **Symmetry verification**: discard or reweight outcomes that violate known conserved quantities.

Mitigation is valuable for NISQ experiments because it can be applied without a full logical-qubit stack. Its cost is sampling overhead and modelling assumptions. PEC in particular can have rapidly growing variance as circuits deepen or noise increases. Thus mitigation is a near-term statistical tool, while correction is the route to scalable reliability.

## From theory to code

`aiquantum.quantum.error_correction` provides a minimum-weight repetition-code decoder and a maximum-likelihood lookup decoder, backing `/api/v1/quantum/error_decode`:

```python
from aiquantum.quantum import decode_repetition_code

recovery = decode_repetition_code([1, 1], n_data=3)  # middle-qubit flip
print(recovery, int(recovery.sum()))                  # weight-1 correction
```

The same module also exposes `build_syndrome_table` and `syndrome_lookup_decoder` for small-code experiments. A useful workflow is to enumerate candidate errors, build a syndrome table, and compare direct lookup with the minimum-weight rule. That is a classical simulation of a code and decoder; it does not by itself execute syndrome extraction on quantum hardware.

### Qiskit-style syndrome circuit sketch

The following Qiskit snippet sketches a three-qubit repetition-code syndrome extraction. It is illustrative: real experiments would add noise models, repeated rounds, reset, scheduling, and backend-specific compilation.

```python
from qiskit import ClassicalRegister, QuantumCircuit, QuantumRegister

# data qubits q[0:3], ancillas a[0:2], classical syndrome bits c[0:2]
q = QuantumRegister(3, "data")
a = QuantumRegister(2, "anc")
c = ClassicalRegister(2, "syn")
qc = QuantumCircuit(q, a, c)

# Example encoded state alpha|000> + beta|111> would be prepared before this point.
# Measure Z0 Z1 into ancilla 0 and Z1 Z2 into ancilla 1.
qc.cx(q[0], a[0])
qc.cx(q[1], a[0])
qc.cx(q[1], a[1])
qc.cx(q[2], a[1])
qc.measure(a[0], c[0])
qc.measure(a[1], c[1])

print(qc)
```

Interpreting the two classical bits requires a convention for which bit value denotes a detection event. In repeated QEC, one usually decodes changes between rounds rather than raw parity values alone.

### Optional Stim-style sketch

Stim is a specialised stabiliser-circuit simulator often used for QEC research. This sketch is optional and uses Stim syntax rather than Qiskit syntax:

```python
import stim

circuit = stim.Circuit("""
R 0 1 2 3 4
CX 0 3
CX 1 3
CX 1 4
CX 2 4
M 3 4
DETECTOR rec[-2]
DETECTOR rec[-1]
""")

samples = circuit.compile_detector_sampler().sample(shots=8)
print(samples)
```

This is still classical simulation: Stim samples an efficient stabiliser model. It is not a claim about a physical device unless calibrated hardware data are supplied separately.

## Summary

Error correction protects quantum information by encoding it into a stabiliser code, extracting syndromes, and decoding the most likely recovery. The repetition code gives the cleanest introduction; the surface code shows why locality, repeated measurements, and graph decoding dominate fault-tolerance discussions. Classical decoders such as minimum-weight matching remain essential baselines. Neural decoders are promising when noise is structured or analogue information is available, but they must satisfy accuracy, robustness, and latency requirements. Error mitigation is a separate NISQ-era strategy for estimating cleaner observables without building logical qubits.
